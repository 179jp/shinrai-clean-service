# 操作ガイド

## 関連サービス
- GitHub ... コード管理サービス（Microsoft）
  - git というソースコードの仕組みを利用したサービス
  - コードの新規追加・更新の履歴を記録する
  - いつ、なぜ、どのファイルのどの部分を変更したか記録して、必要に応じて戻したり、進めたりできる
- Netlify(Cloudflare Pages) ... webサイトをホストするサービス
  - GitHub の変更を検知して、コードからwebサイトとして見える状態に組み立て（ビルド）、サーバーに載せる（デプロイ）する
  - ユーザーがサーバーを借りなくても、クラウドサーバー（AWSなど）を内部的に使用してデプロイしている
  - 個人ユーザー程度の利用なら、運用費がかからずホストできる
- お名前.com ... ドメイン取得サービス
  - 取得したドメインを DNS 設定を行うことでサーバーやサービスに紐付ける

### 参考
- なぜビルドが必要か
  - テンプレートを使用して、よく使う部分の部品化やブログ・更新管理しやすい仕組みにしている
  - GitHub に載せているのは共通パーツを含まない設計図のみで、最終サイトの形にするためには組み立てが必要
- git / GitHub
  - 変更履歴は Tree のイメージ
    - 変更を加えたらコミットして記録する
    - ブランチ ... 変更を加える際に、本番には影響を与えないように作業スペースを用意できる機能
      - 変更して、問題なければ、作業スペースでの変更内容を本番に取り込む（マージ）ことができる
      - 変更内容を本番に取り込んで問題ないか、確認する機能としてプル・リクエストがある
      - 変更したが、問題がある or 今ではないなどの場合はマージしなければ、本番には影響を与えない
    - 今回のサイトでのブランチ
      - `main` ... 本番
  - 間違えて変更を加えても、履歴をたどり、戻す・復元が可能なので、安心して変更できる

### 用語集
|用語|例|説明|
|:---|:---|:---|
|リポジトリ|`shinrai-clean-service`|GitHubに作れるサイト・プロジェクトなど|
|コミット||変更をgitに記録すること|
|ブランチ|`main`|本番には影響を与えないように作業スペースを作成する機能|
|プル・リクエスト（Pull Request / PR）||変更内容を取り込んで問題ないか、確認する機能|
|マージ||変更内容を取り込むこと|

## LLMでの更新手順

### 前提
- Codex / Claude Code での使用を想定
  - Codex の方がローカルに環境を作らなくても使用できるので簡単
- Codex / Claude Code は GitHub からコードを読み込み、必要な箇所を変更
  - Codex は設定したブランチ（`main`がおすすめ）から、作業用ブランチを作成し、作業する
  - 変更完了したらプル・リクエストが作成される

### 準備
以下は一度設定したら問題なし

1. Codex で GitHub のリポジトリ `shinrai-clean-service` と接続
2. nodeのバージョンは新しく設定しておく

### 更新手順

1. Codexを[web](https://chatgpt.com/codex/cloud) or アプリで開く
2. リポジトリは `shinrai-clean-service`、ブランチは `main` を選択
3. Chatベースで変更を依頼する
  - できるだけ変更箇所など具体的に指示する方が間違いが起きにくい
    - NG: `税理士にルビを振って`
    - OK: `Messageの本文の中の税理士にルビを振って`
  - 一度に大きな変更を渡すよりは、小分けにして渡す方が正確に実施されやすい
  - 「PRを作成する」を押すまでは、どこにも反映されない状態
  - もし大きめの変更を依頼するとき（既存パーツにないものを追加するなど）は以下のコメントを入れてもらえると、サイト内の破綻リスクが避けられると思います
    - 「※既存のコードをよく読み、`Documents/Design.md`の原則に沿って変更を加えること」

4. 変更ができたと思ったら「PRを作成する」ボタンを押すかchatで依頼する
5. 「PRを作成する」ボタンが「PRを表示する」ボタンに変わるので押す → GitHubへリンク
6. Cloudflare で自動ビルドが走る（本番ではなくプレビュー用）
7. ビルドが完了し、緑になったら view details を押して、表示されるプレビューURLを押して、変更を確認する
8. 変更内容に修正がある場合は Codex のチャットに戻り、修正点を指摘、変更する
9. GitHub のマージボタンを押せば、変更が`main`ブランチに取り込まれ、本番サイトにも反映される

- もし、変更依頼したものの「これはダメだ」という時は、PRをクローズすれば問題なし
